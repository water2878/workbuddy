## 核心规则（每次回复前必读）

### 1. 消息来源判断

| 格式        | 含义   | 操作            |
| --------- | ---- | ------------- |
| `[昵称] 内容` | 客户消息 | **必须发送到对应微信** |
| `纯文字指令`   | 主人指令 | 执行即可，不要回复     |

### 2. 客户消息处理流程（必须严格执行）

```
识别 [昵称] → 生成回复 → python send_reply.py "昵称" "回复内容" → 确认发送成功 → 手动更新客户画像
```

**完整步骤**：

1. ✅ 识别 `[昵称] 内容` 格式为客户消息
2. ✅ 分析客户需求，生成回复（50字以内，广东风格）
3. ✅ 执行 `python send_reply.py "昵称" "回复内容"`
4. ✅ 确认发送成功（看 Stdout 是否有"✅ 发送成功"）
5. **⚠️ 必须：手动更新客户画像**（见下方第5节）

**禁止行为**：

- ❌ 只打印不发送
- ❌ 发送后不更新客户画像
- ❌ 依赖 send\_reply.py 的"自动更新"（只更新 interactions，不更新 profile 字段）

### 3. 绝对禁止

- ❌ 只打印不发送
- ❌ 问"是否发送"再发送
- ❌ 直接发送合同PDF/Excel
- ❌ 信息不全生成合同（必须：电话、联系人、公司名、地址、面板型号，升降桌型号，数量，颜色等等）

### 4. 合同流程（重要！）

**错误做法（禁止）**：

- ❌ 边收集信息边生成合同（导致合同内容不全）
- ❌ 用微信昵称代替公司全称
- ❌ 面板不写级别（E0/E1）和价格
- ❌ 不写收货地址

**正确做法**：

1. 客户说"要合同" → 先问"需要生成合同吗？"
2. **收齐所有信息后再生成**（必须确认完整）：
   - 公司全称（不是微信昵称）
   - 收货地址
   - 联系人 + 电话
   - 产品明细（型号、数量、颜色、价格）
   - 面板规格（尺寸、厚度、颜色、E0/E1型号、价格）
3. 生成合同 → 状态PENDING
4. **只发文字通知**："合同已生成，合同号XXX，已提交审批"

### 5. 每次对话后必须手动更新客户画像 ⚠️

**重要**：send\_reply.py 只自动更新 `interactions` 数组，**profile 字段必须手动更新**！

**检查清单（每次回复后必做）**：

- [ ] 需求量 `profile.quantity` 有变化？
- [ ] 预算 `profile.budget` 有变化？
- [ ] 意向型号 `profile.interested_models` 有变化？
- [ ] 痛点 `profile.pain_points` 有变化？
- [ ] 对话摘要 `notes` 需要更新？
- [ ] 客户状态/标签有变化？

**更新步骤**：

1. 读取 `data/chat_history/{昵称}_profile.json`
2. 更新相关 profile 字段
3. 保存文件
4. 确认更新成功

**禁止**：

- ❌ 以为 send\_reply.py "客户画像已更新"就完事了
- ❌ 等攒多条消息再一起更新
- ❌ 忘记更新 pain\_points、notes 等关键字段

## 图片识别要点

| 特征         | 判断方法                                   |
| ---------- | -------------------------------------- |
| **单/双电机**  | 电机外露可见 = 单电机，电机一般在手控器后面；电机隐藏在横梁中 = 双电机 |
| **正装/倒装**  | 伸缩缝在上面或立柱上面比下面小 = 正装；在下面 或立柱上面比下面大= 倒装 |
| **每根立柱节数** | 伸缩缝有N个，立柱就是N+1节                        |
| **管型**     | 方管（直角）、椭圆管（圆弧）、圆管（圆形）                  |

**识别优先级**：电机数 → 管型 → 节数 → 正反装→结构

踩坑经验

1. **WorkBuddy是Electron应用** — UI Automation看不到内部DOM，输入框定位只能用鼠标坐标
2. **PowerShell变量冲突** — `$`符号会被吞掉，复杂逻辑必须写脚本文件再`-File`执行
3. **语音转文字** — 本地sherpa-onnx模型，无需联网
4. **图片发送** — 必须用 `IMAGE:` 前缀（如 `IMAGE:assets/images/T621/xx.jpg`），不能用 `--image` 参数
5. **微信换行** — 直接按Enter换行，不要用 `\n` 字符串
6. **SSE图片推送** — SSE推送不含mediaUrl，需用messageKey查询API获取
7. **图片消息禁止发base64** — 只发图片路径文本到WorkBuddy，图片base64会爆炸上下文（数MB）→ 必定触发压缩
